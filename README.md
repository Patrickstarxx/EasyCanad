# EasyCanad — 鸭翼飞行器混控控制器

基于 Arduino 的鸭翼（canard）飞行器控制系统，支持 AOA 攻角传感器反馈、双路 PWM 遥控输入、俯仰/副翼混控输出，以及 EEPROM 零位校准。

## 功能特性

- **AOA 反馈稳定**：通过模拟攻角传感器闭环控制鸭翼，对准来流
- **双路 RC 输入**：俯仰（升降）+ 副翼，各自独立中断采集
- **俯仰/副翼混控**：俯仰 → 两鸭翼同向偏转；副翼 → 两鸭翼差动偏转
- **一键零位校准**：按下按钮即记录当前攻角为零位，保存到 EEPROM
- **200Hz 控制刷新**：5ms 控制周期，一阶低通滤波平滑传感器数据

## 硬件连接

### 引脚接线表

| Arduino 引脚 | 连接设备 | 说明 |
|-------------|---------|------|
| **A0** | AOA 攻角传感器 | 模拟输入 0~5V |
| **D2** | RC 接收机 — 俯仰(升降)通道 | PWM 输入，INT0 中断 |
| **D3** | RC 接收机 — 副翼通道 | PWM 输入，INT1 中断 |
| **D4** | 标定按钮 | 按下接 GND（INPUT_PULLUP） |
| **D9** | 左鸭翼舵机 | PWM 输出 500~2500μs |
| **D10** | 右鸭翼舵机 | PWM 输出 500~2500μs |
| **5V / GND** | 电源总线 | 给 Arduino、传感器、舵机供电 |

### 接线示意

```
                        ┌──────────────────┐
                        │    Arduino       │
                        │                  │
  AOA 传感器 ────────── │ A0           D9  │ ─── 左鸭翼舵机
                        │              D10 │ ─── 右鸭翼舵机
  RC 俯仰通道 ───────── │ D2 (INT0)        │
  RC 副翼通道 ───────── │ D3 (INT1)        │
  标定按钮 ──────────── │ D4              │
                        │                  │
  5V ────────────────── │ 5V           GND │ ─── GND
                        └──────────────────┘
```

> **注意**：舵机供电建议使用独立 BEC，不要从 Arduino 5V 引脚取电，否则大电流可能导致复位。

## 编译与烧录

1. 用 **Arduino IDE** 打开 `EasyCanad.ino`
2. 选择开发板：**Arduino Uno**（或 Nano，ATmega328P）
3. 确保 `Servo.h` 和 `EEPROM.h` 库已安装（Arduino IDE 自带，无需额外安装）
4. 点击 **上传**

## 调试

### 串口监视器

烧录后打开串口监视器（115200 baud），约每 100ms 输出一行：

```
AOA=-3.25, Pit=1500(0.0), Ail=1498(-0.9), L=-3.3(1467), R=-2.4(1476)
```

各字段含义：

| 字段 | 含义 | 单位 |
|------|------|------|
| `AOA` | 滤波后的攻角（相对零位） | deg |
| `Pit` | 俯仰通道原始脉宽（括号内为叠加角度） | μs (deg) |
| `Ail` | 副翼通道原始脉宽（括号内为叠加角度） | μs (deg) |
| `L` | 左鸭翼最终指令角度（括号内为输出脉宽） | deg (μs) |
| `R` | 右鸭翼最终指令角度（括号内为输出脉宽） | deg (μs) |

### 常见调参场景

| 现象 | 调整方式 |
|------|---------|
| 鸭翼抖动、响应太敏感 | 减小 `aoaFilterAlpha`（例如 0.15 → 0.08） |
| 遥控打杆舵面动作太小 | 增大 `rcAuthorityDeg` |
| 副翼滚转力度不够 | 增大 `rollAuthorityDeg` |
| 鸭翼偏转超出机械极限 | 减小 `canardLimitDeg` |
| 副翼方向反了 | 将 `rollCmd = ailAddDeg` 改为 `rollCmd = -ailAddDeg` |

## 标定流程

1. 将飞行器放置在**实际飞行的零攻角姿态**（即正常巡航姿态）
2. 按下标定按钮（D4 接 GND）
3. 串口输出 `CAL saved. zero(deg)= xx.xxx`
4. 零位已写入 EEPROM，断电不丢失

## 混控算法

```
pitchCmd = -1.2 × AOA(滤波) - 俯仰RC输入
rollCmd  = 副翼RC输入 × rollAuthorityDeg

左鸭翼 = pitchCmd + rollCmd   ← 俯仰 + 滚转
右鸭翼 = pitchCmd - rollCmd   ← 俯仰 - 滚转

输出脉宽 = map(角度, -90°, 90°, 500μs, 2500μs)
```
